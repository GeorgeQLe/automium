# Prototype Session Loop Convention — moved

> **This convention has moved.** It is superseded by and absorbed into
> **`docs/design-tree-loop-convention.md`** (the Design Tree Loop Convention), which unifies the
> design-phase product-design and product-testing skills onto one tree-growing, 5-stage
> orchestrator pattern (`interrogation → research → design → plan → implement(scoped)`).

The intra-skill substep chunking + shared-context-brief mechanics that used to live here are
now **§2 "The Universal 5-Stage Per-Skill Session Flow"** of the new document. The cross-skill
phase map, routing rules, approval rules, task rules, and artifact rules are now §§1–8 there,
and §0 relates this design loop to the Research Session Loop as two instances of one
meta-pattern.

- Per-skill bundles are now `DESIGN-TREE-LOOP.md`, generated by
  `scripts/upgrade-design-tree-loop.mjs` (this doc's old generator,
  `scripts/upgrade-prototype-session-loop.mjs`, is retired).
- `docs/research-session-loop-convention.md` (business-research Pattern A) is a separate,
  unchanged-but-cross-referenced convention and was never part of this one.

**Go to `docs/design-tree-loop-convention.md`.**
